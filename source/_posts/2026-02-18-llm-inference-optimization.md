# 大模型推理优化：KV Cache、量化、并发与成本

## 1. 先把指标量出来

优化前先统一看四个指标：

- 首 token 延迟（TTFT）
- token/s
- 显存占用
- 单次请求成本（长度、并发、缓存命中）

## 2. KV Cache：为什么它这么重要

在自回归生成里，KV Cache 可以避免重复计算历史 token 的注意力，从而显著降低后续 token 的计算量。

工程上你要关注：

- 上下文越长，KV Cache 越“值钱”
- 并发越高，KV Cache 占用越明显

## 3. 量化：速度、精度与稳定性的三角形

- 低比特量化通常更快、更省显存
- 但可能引入输出不稳定、对数字/代码更敏感的退化

建议：对你最核心的任务建立回归集，量化切换必须过回归。

## 4. 并发策略：别把服务打爆

- 先限制最大并发与最大输出长度
- 再考虑批处理（batching）
- 对长请求和短请求做队列隔离

## 5. 可操作的排查顺序

1. 限制 max_tokens，确认是否是“生成太长”导致慢
2. 检查是否启用了 GPU/加速后端
3. 开启/验证 KV Cache
4. 尝试量化并对回归集对比
5. 设计并发队列与熔断策略

推理优化的本质是：把性能变成可观测指标，再用最小改动换最大收益。
